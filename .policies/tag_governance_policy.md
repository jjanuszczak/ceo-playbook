# Tag Governance Policy

This document establishes the official rules and taxonomy for site tags. It is based on [Issue #2](https://github.com/jjanuszczak/ceo-playbook/issues/2).

## 1. The Prime Directive (Non-Negotiable)

> **Every tag must be reusable and meaningful at scale.**

If you cannot confidently imagine **10+ posts** using a tag over time, it does **not** deserve to exist.

## 2. Tag Purpose

Tags exist to:
1.  Create **cross-category connections**
2.  Express **how you think**, not just what you write about
3.  Enable **semantic discovery**
4.  Reinforce **long-term intellectual threads**

## 3. Tag Types

Use tags from these four buckets:

### A. Conceptual Tags (Primary)
*   **Rules:** Must appear across categories. Defines the intellectual spine.
*   *Examples:* `platform-economics`, `systems-thinking`, `capital-allocation`.

### B. Domain Tags
*   **Rules:** Specific to a field. Avoid duplicating category names.
*   *Examples:* `embedded-finance`, `energy-markets`, `electric-vehicles`.

### C. Practice / Operator Tags
*   **Rules:** How work gets done. Valuable for practitioners.
*   *Examples:* `venture-building`, `board-governance`, `go-to-market`.

### D. Thinker / Source Tags (Limited)
*   **Rules:** Recurring intellectual influences. No one-offs.
*   *Examples:* `kevin-kelly`, `ben-thompson`.

## 4. Hard Constraints

*   **Count:** Ideal 5–8 tags per post (Max 10, Min 3).
*   **Format:** Lowercase, hyphenated, singular (`long-term-thinking`, `ecosystem`).
*   **No-Go List:** `ai` (use `artificial-intelligence`), `tech`, `thoughts`, `misc`, `general`, `trends`.

## 6. The "New Tag" Test

Before creating a new tag, ask:
1.  Will I reuse this **at least 5 times** in 12–18 months?
2.  Does this express a **core idea**, not a detail?
3.  Could this replace an existing tag with better clarity?
4.  Would I want a **landing page** for this tag?

**If any answer is "no" → do not create the tag.**
